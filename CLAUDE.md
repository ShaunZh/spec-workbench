# CLAUDE.md

## Project Overview

**Project Name:** ReqPilot  
**Project Type:** Product project + learning project  
**Current Stage:** MVP / Day 1  
**Development Mode:** Developer-led, AI-assisted

ReqPilot is an **AI Requirement Analysis Workbench**.  
It helps users turn raw requirement text, bug descriptions, or feedback into structured analysis results.

This project also serves as a **hands-on learning vehicle for AI agent development**.  
The project is intentionally built through **developer + AI collaboration**, so the repository should optimize for:

- fast scaffolding
- clear structure
- easy review
- incremental learning
- minimal over-engineering

---

## Product Goal

Build a web-based AI workbench that supports:

- chat-based interaction
- analysis mode for requirement text
- structured outputs
- visible tool calling
- conversation history
- Markdown export

Target structured output fields:

- `summary`
- `todos`
- `risks`
- `acceptance_criteria`
- `open_questions`

---

## Learning Goal

This project is also used to learn and practice:

- LLM application development
- structured outputs
- tool calling
- basic single-agent workflow design
- prompt design and iteration
- AI-assisted development workflow

The project should not only produce a working demo, but also help the developer understand:

- what the model does
- what the tool does
- how workflow is orchestrated
- what should be generated by AI

## Architecture Principles

Follow these principles strictly:

1. **MVP first**
   - Prefer the smallest working version.
   - Avoid premature abstraction.

2. **Do not over-engineer**
   - No unnecessary patterns, layers, services, or frameworks.
   - No microservices.
   - No enterprise architecture.

3. **Keep code readable**
   - Clear naming
   - Small files where reasonable
   - Simple data flow

4. **Prefer explicit over clever**
   - Avoid magic abstractions.
   - Prefer straightforward implementations.

5. **Build in layers**
   - first: skeleton
   - then: basic CRUD
   - then: AI integration
   - then: workflow
   - then: UX polish

## AI Collaboration Rules

AI is used as a coding assistant, not as the project owner.

### AI should help with
- project scaffolding
- directory structure
- boilerplate code
- basic CRUD routes
- simple schemas and models
- config files
- README drafts
- mock data
- repetitive refactors
- code cleanup suggestions

- product scope
- data model tradeoffs
- structured output schema design
- prompt design
- tool definition
- workflow logic
- interaction design
- edge cases
- error handling policy
- architecture tradeoffs

If AI proposes something beyond current scope, keep the simpler option.

## Coding Guidance

### General
- Write minimal, working, readable code.
- Avoid speculative features.
- Avoid introducing dependencies unless requested.
- Prefer native framework capabilities.

### Frontend
- Keep components simple and composable.
- Do not introduce complex state libraries on Day 1.
- Do not use third-party UI component libraries unless asked.
- Prefer clean layout and placeholder UI over fake complexity.

### Backend
- Use FastAPI with `APIRouter`.
- Keep route structure clean.
- Do not add service/repository patterns unless needed later.
- Keep database setup minimal.

### Database
- Use SQLite-compatible definitions.
- Keep models minimal.
- Do not optimize early.
- Favor simple schema over future-proofing.
