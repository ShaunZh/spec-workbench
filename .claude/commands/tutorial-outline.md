请把我提供的材料整理为一个 **可持续扩展的教程大纲**。

处理要求：
1. 先识别这批材料处于 AI agent 学习路径中的哪个阶段。
2. 提炼出这一阶段真正完成了什么，而不是理想上应该完成什么。
3. 不要只复述笔记，要把内容转换成面向教学的章节安排。
4. 如果材料中包含失败尝试、报错、调试过程，要将其转化为“读者会遇到的问题”。
5. 如果材料不足以支撑完整大纲，要明确标出缺口。

输出格式必须使用下面结构：

# 系列标题

## 目标读者

## 这个阶段的学习目标

## 当前阶段摘要
- 用 3-6 条概括这批材料实际上覆盖了什么

## 教程章节建议
1. 章节标题
   - 本章解决的问题：
   - 核心内容：
   - 与前后章节的关系：

## 已经具备的素材
- 列出可以直接写成教程的内容

## 明显缺失的素材
- 列出还缺哪些信息、案例、代码或验证结果

## 推荐下一步
- 建议接下来优先补哪一章，为什么

额外要求：
- 默认读者是“会 Python，但没系统做过 AI agent 的开发者”。
- 语言保持清晰，不要写成营销文案。
- 如果我输入的是零散笔记，请主动做主题聚类。
